Readme file : # PROJECT RULES

## Project Name

LegalEase — Online Lawyer Hiring Platform

---

## Project Goal

Build a recruiter-level SaaS application using Next.js and Express.

The project must be production-ready, scalable, maintainable, responsive, and visually premium.

---

# Tech Stack

## Frontend

- Next.js (Latest App Router)
- JavaScript
- Tailwind CSS
- HeroUI
- Framer Motion

## Backend

- Express.js
- MongoDB Native Driver
- Better Auth
- JWT
- Stripe
- imgBB

---

# UI Rules

- Premium SaaS UI
- Minimal
- Modern
- Clean
- Professional
- Glass effects only where necessary
- Large white space
- Soft shadows
- Rounded corners
- Consistent spacing

---

# Typography

Heading → Syne

Body → Poppins

---

# Component Rules

Always create reusable components.

Keep components small.

Never duplicate UI.

---

# Coding Rules

- Clean code
- Readable code
- Proper folder structure
- Meaningful names
- Reusable functions
- Responsive first
- Accessibility first

---

# Development Workflow

1. Plan
2. Design
3. Build
4. Review
5. Refactor
6. Commit
7. Test

---

# Git Rules

Meaningful commits only.

Never commit broken code.

---

# Final Goal

Create a production-quality portfolio project that impresses recruiters and is easy to explain during technical interviews.
